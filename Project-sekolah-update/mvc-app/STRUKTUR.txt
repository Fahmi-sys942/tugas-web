ALUR KERJA MVC
==============

1. USER REQUEST
   ↓
2. index.php (Front Controller)
   - Menerima semua request
   - Routing ke controller yang tepat
   ↓
3. CONTROLLER
   - Menerima input dari user
   - Validasi data
   - Memanggil MODEL
   ↓
4. MODEL
   - Berinteraksi dengan database
   - Memproses data
   - Mengembalikan hasil ke CONTROLLER
   ↓
5. CONTROLLER
   - Menerima data dari MODEL
   - Mengirim data ke VIEW
   ↓
6. VIEW
   - Menerima data dari CONTROLLER
   - Render HTML dengan data
   ↓
7. RESPONSE ke USER


ROUTING SYSTEM
==============

URL Pattern: index.php?page=[PAGE]&action=[ACTION]&id=[ID]

Contoh:
- index.php?page=home                           → Dashboard
- index.php?page=login                          → Halaman Login
- index.php?page=login&action=logout            → Logout
- index.php?page=mahasiswa                      → List Mahasiswa
- index.php?page=mahasiswa&action=create        → Form Tambah Mahasiswa
- index.php?page=mahasiswa&action=edit&id=1     → Form Edit Mahasiswa
- index.php?page=mahasiswa&action=delete&id=1   → Hapus Mahasiswa
- index.php?page=products                       → List Products
- index.php?page=products&search=laptop         → Search Products


DATABASE SCHEMA
===============

Table: users
- id (INT, PK, AUTO_INCREMENT)
- username (VARCHAR(50), UNIQUE)
- password (VARCHAR(255))
- role (ENUM: 'admin', 'user')

Table: mahasiswa
- id (INT, PK, AUTO_INCREMENT)
- nim (VARCHAR(20), UNIQUE)
- name (VARCHAR(100))
- jurusan (VARCHAR(100))
- foto (VARCHAR(255), NULLABLE)
- alamat (TEXT)
- created_at (TIMESTAMP)
- updated_at (TIMESTAMP)

Table: products
- id (INT, PK, AUTO_INCREMENT)
- name (VARCHAR(100))
- price (DECIMAL(10,2))
- created_at (TIMESTAMP)
- updated_at (TIMESTAMP)
